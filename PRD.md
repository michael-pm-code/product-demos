# Subotiz A/B 测试管理后台 - 产品需求文档（PRD）

**版本**：1.0  
**目标读者**：开发、UI/UX 设计师  
**用途**：指导 A/B 测试管理后台的设计与开发实现  

---

## 一、文档说明

### 1.1 产品背景

A/B Test 系统用于帮助商户通过创建实验验证不同策略对转化率的影响，优化定价/价格表等配置，提升转化率与收入。本 PRD 描述商户端管理后台的全部功能、交互与业务规则。

### 1.2 功能范围

- **管理后台**：实验的创建、编辑、状态管理、用户分组管理、数据查看与 AI 解读。
- **前端 SDK**（不在本文档）：为访客生成唯一标识、实验分组、拉取实验规则、返回变量值；商户在站点集成 SDK 后根据返回值做不同策略处理。

### 1.3 术语

| 术语 | 说明 |
|------|------|
| 实验对象 | 实验作用的目标类型：定价、价格表、或自定义变量 |
| 对照组 | 不施加改动的分组，用于与实验组对比 |
| 实验组 | 施加不同策略的分组（如不同定价、不同文案等） |
| 变量名/实验变量名 | 自定义实验时，前端通过该 key 向 SDK 获取当前用户的分组对应值 |

## 二、全局与导航

### 2.1 布局结构

- **左侧导航**：固定侧栏，一级菜单为「数据」「商品」。
- **主内容区**：顶部页面标题（可带左侧返回按钮），下方为路由内容。

### 2.2 导航结构

| 一级菜单 | 二级菜单 | 是否可点击 | 路由/说明 |
|----------|----------|------------|-----------|
| 数据 | 交易概览 | 否 | 仅展示，对齐现有后台 |
| 数据 | 订阅概览 | 否 | 仅展示 |
| 数据 | 数据报表 | 否 | 仅展示 |
| 数据 | **A/B 测试** | 是 | 实验列表页 |
| 商品 | 商品 | 是 | 商品概览页 |
| 商品 | 定价 | 是 | 定价列表页 |
| 商品 | 价格表 | 是 | 价格表列表页 |

- 当前路由对应的二级菜单项高亮；所属一级菜单标题在高亮时也可做视觉强调。

### 2.3 页面标题与返回

- **A/B 测试列表**：标题「A/B 测试」，无返回按钮。
- **创建实验**：标题「创建实验」，标题左侧有「← 返回」按钮。
- **实验详情**：标题「实验详情」，标题左侧有「← 返回」按钮，点击返回实验列表。
- **商品 / 定价 / 价格表**：标题为对应名称，无返回按钮。

**创建实验页返回逻辑**：

- 若表单**未填写任何有效内容**：点击返回直接进入实验列表。
- 若表单**已填写内容**：点击返回弹出确认弹窗。  
  - 文案：「退出后填写的内容将丢失，是否继续？」  
  - 按钮：「取消」（关闭弹窗）、「继续退出」（关闭弹窗并进入实验列表）。

「已填写」判定范围建议：实验名称、实验目标、实验对象、实验周期、实验变量名（自定义时）、定向范围标签、任一分组的定价/价格表/自定义实验对象值。

## 三、A/B 测试列表页

### 3.1 列表有数据时

- **表格列**（从左到右）：  
  实验名称、**实验 id**、创建时间、创建人、实验对象、实验时长、实验用户量、状态。

- **实验名称列**：可展示名称 + 副文案（如定价/价格表名称或链接），支持多行。
- **实验 id 列**：展示实验唯一标识（如 `exp-001`），便于用标识定位问题。

**复制能力**（用于快速用实验标识定位问题）：

- 「实验名称」「实验 id」两列：鼠标悬停时在单元格内右侧展示「复制」按钮。
- 点击「复制」：将当前列对应内容写入剪贴板（名称列复制实验名称，id 列复制实验 id）。
- 复制成功后按钮可短暂变为「已复制」，约 1.5 秒后恢复为「复制」。
- 点击复制按钮不触发整行点击（不进入详情）。

- **状态**：展示为标签，可选状态见 3.3。
- **操作**：点击整行（除复制按钮外）进入该实验详情页。
- **右上角**：常驻「+ 创建实验」按钮，进入创建实验页。

### 3.2 列表无数据时

- 列表区域展示**空状态**：
  - 图标 + 标题「还没有任何实验」。
  - 说明文案：创建第一个 A/B 实验，验证定价或价格表配置对转化率与收入的影响。
  - 「创建实验」按钮。
- 空状态下方仍展示「缺省状态」模块（见第七章），引导理解功能并创建实验。

### 3.3 实验状态枚举

| 状态值 | 展示文案 | 说明 |
|--------|----------|------|
| draft | 草稿 | 刚创建或未激活 |
| active | 运行中 | 正在放量 |
| paused | 已暂停 | 已暂停放量，可恢复 |
| ended | 已结束 | 实验结束 |

## 四、创建实验页

### 4.1 表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 实验名称 | 单行文本 | 是 | 供商户在后台区分实验，如「定价页标题 A/B 测试」 |
| 实验目标 | 单行文本 | 否 | 如「提升定价页下单转化率」 |
| 实验对象 | 单选 | 否 | 选项：定价、价格表、**自定义** |

**实验对象 = 定价 或 价格表时**：

- 不再在实验对象下展示二级下拉（不在此处选择具体定价/价格表）。
- 在「实验分组与流量」模块中，中间列表头为「定价」或「价格表」，每行为该分组选择的定价或价格表（下拉选择，选项来自业务数据）。
- 对照组行下方展示提示：「商户站点前端集成的价格或价格表需要与对照组保持一致。」

**实验对象 = 自定义时**：

- 在「实验对象」下方增加：
  - **实验变量名**：单行输入，占位示例「如：checkout_style…」。
  - 说明文案：「前端在实验节点调用 SDK 方法获取用户在此变量的实验分组数据，商户根据返回值做出不同实验策略处理。」
  - 文案后增加文字链接「了解详情」，点击在**新标签页**打开「自定义 A/B Test 实验配置说明」页（见 4.5）。
- 在「实验分组与流量」中，中间列表头为「**自定义实验对象值**」，每行为文本输入，商户为各分组填写要下发给前端的值（如 `default`、`variant_a`）。
- 该模块下方展示提示：「集成 SDK 后，在需要实验的节点通过 SDK 获取该变量名的值，SDK 将按用户分组返回对应列填写的值，请在前端根据返回值自行处理业务逻辑。」

| 实验周期 | 开始日期、结束日期 | 否 | 日期选择器 |
| 实验用户范围 | 单选 + 条件配置 | 否 | 选项：全部用户、定向范围。定向时可添加多条规则，维度如：国家、来源渠道、设备；取值为文本或多选等（以业务为准） |
| 实验分组与流量 | 表格 + 操作 | - | 见 4.2 |

### 4.2 实验分组与流量

- **表头**：组名称 | （定价 / 价格表 / 自定义实验对象值） | 流量 %  
  - 第三列表头随「实验对象」变化：选定价为「定价」，选价格表为「价格表」，选自定义为「自定义实验对象值」。
- **行**：
  - 第一行固定为**对照组**，组名称可只读。
  - 可再添加 1～5 个**实验组**（即总组数 2～6）。
  - 每行：组名称（对照组只读）、中间列（定价/价格表为下拉，自定义为文本输入）、流量 %（数字输入）。
- **流量规则**：
  - 各分组流量百分比之和须为 100%。
  - **添加实验组**时：新增一行，并将**所有分组**的流量自动重新均分，保证总和 100%（如 4 组则 25%、25%、25%、25%）。
  - 商户可手动修改各组分比，前端可做 100% 校验与提示。
- **对照组提示**：仅在实验对象为定价或价格表时显示，见 4.1。

### 4.3 提交与跳转

- **保存为草稿**：提交后实验状态为「草稿」，并跳转回实验列表（或提示成功并留在当前页，以产品为准）。
- **取消**：若有未保存内容，建议与「返回」一致做二次确认（可选）。

### 4.4 从定价/价格表页带入

- 在「定价」或「价格表」列表中，每行提供「创建 A/B 实验」按钮。
- 点击后跳转到创建实验页，并**自动**：
  - 将「实验对象」设为对应类型（定价 或 价格表）。
  - 将**对照组**的中间列（定价/价格表）设为当前行对应的定价或价格表。

### 4.5 自定义 A/B Test 实验配置说明页

- **入口**：创建实验页 - 实验对象为「自定义」时，变量名说明旁的「了解详情」。
- **打开方式**：新标签页打开，路由建议独立（如 `/docs/custom-abtest-config` 或 hash 等价）。
- **内容**：说明文档，包含但不限于：
  - 如何配置实验变量名；
  - 如何为各分组填写「自定义实验对象值」；
  - 前端如何在实验节点调用 SDK、根据返回值做不同策略。

## 五、实验详情页

### 5.1 页面结构

- **顶部**：实验名称 + 操作区（见 5.2）。
- **Tab**：**实验详情 / 管理**（默认）、**实验数据**。切换 Tab 不离开当前实验。

### 5.2 头部操作区（按状态展示）

- **编辑配置**  
  - 仅当状态**不是**「已结束」时展示。  
  - 点击进入编辑（表单与创建实验一致，可修改名称、目标、对象、周期、用户范围、分组与流量等）。

- **状态标签**  
  - 始终展示：草稿 / 运行中 / 已暂停 / 已结束。

- **已结束**  
  - 当状态为「已结束」时，在状态标签**左侧**展示非强调文案：  
    **「实验于 yyyy-mm-dd hh:mm:ss 已结束」**  
  - 时间来自该实验的结束时间字段（如 `endedAt`）。若无则展示「—」或占位。

- **草稿**  
  - 展示按钮「**激活实验**」。点击后将状态改为「运行中」，并刷新页面。

- **已暂停**  
  - 展示按钮「**继续实验**」。点击后将状态改为「运行中」，并刷新页面。

- **运行中**  
  - 展示「**操作**」下拉按钮，选项：
    - **暂停实验**  
    - **结束实验**  
  - 点击「暂停实验」：弹出确认弹窗（见 5.3），确认后将状态改为「已暂停」，并记录暂停时间（若有字段）。
  - 点击「结束实验」：弹出确认弹窗（见 5.3），确认后将状态改为「已结束」，并记录结束时间（如 `endedAt`）。

### 5.3 暂停/结束确认弹窗

- **暂停实验**  
  - 文案：「暂停后将不再放量，不影响已参与实验用户。」  
  - 按钮：「取消」「暂停实验」。

- **结束实验**  
  - 文案：「自定义对象的实验需要注意处理实验结束的逻辑，避免影响用户体验。」  
  - 按钮：「取消」「结束实验」。

- 确认后执行对应状态变更并关闭弹窗，刷新详情内容。

### 5.4 Tab：实验详情 / 管理

- **实验配置信息**（只读展示）：  
  实验目标、实验对象（类型 + 名称/ID 或链接）、实验周期、实验用户范围（全部或定向规则列表）、实验分组与流量（组名 + 流量占比）。

- **用户实验分组数据**  
  - **表格列**：用户 Uid、用户 Vid、用户分组、分组时间。  
  - **搜索**：表格上方右侧提供搜索框，占位「搜索用户 Uid / Vid」，支持按 Uid 或 Vid 模糊查询（不区分大小写），用于测试时快速定位用户。  
  - **分页**：数据量较大时分页展示（如每页 10 条），底部展示「共 x 条」及上一页/下一页（或页码），切换分页时保留搜索条件。  
  - **修改分组**：每行「用户分组」为下拉框，商户可手动将某用户改为其他分组，保存后生效（具体保存策略由后端约定）。

### 5.5 Tab：实验数据

- **汇总**：实验时长、参与用户数（可选更多指标）。
- **转化漏斗（5 层）**  
  - 指标顺序：进入页面数 → 发起结账数 → 提交订单数 → 完成支付操作数 → 支付成功数。
- **表格规则**  
  - 每行一个实验组（对照组、实验组 A、实验组 B…）。  
  - 列顺序：每个指标列后紧跟**对应的转化率列**（表头需与数据列一一对应）：
    - 发起结账数 → **发起结账率**（= 发起结账数 / 进入页面数）
    - 提交订单数 → **提交订单转化率**（= 提交订单数 / 发起结账数）
    - 完成支付操作数 → **完成支付操作转化率**（= 完成支付操作数 / 提交订单数）
    - 支付成功数 → **支付成功转化率**（= 支付成功数 / 完成支付操作数）
    - 最后一列 → **整体转化率**（= 支付成功数 / 进入页面数）
  - 转化率数值默认无强调色；**实验组**的转化率下方可展示小字：相对对照组的增长百分比，正数用绿色、负数用红色。
  - 各转化率列表头支持 **hover 展示计算方式**（如 tooltip：「计算方式：发起结账数 ÷ 进入页面数」等）。
- **AI 数据解读**（可选）  
  - 表格下方增加「AI 数据解读」模块，对当前实验漏斗数据做简要解读，帮助商户理解结果。具体文案与接口由产品与算法约定。

## 七、缺省状态模块与说明文案

### 7.1 缺省状态模块（无实验时的引导）

- **出现时机**：A/B 测试列表页；无实验时在空状态下方展示，有实验时也可在列表下方保留一块用于演示/引导（以产品为准）。
- **内容**：  
  - 上方：**插图**，能传达「实验」与「转化率提升」含义（如趋势线 + A/B 分组示意）。  
  - 中间：**文案**——「A/B Test 系统可以创建实验来验证不同策略对转化率的影响，帮助您优化配置，提升转化率。」其中「提升转化率。」不换行（如用 `white-space: nowrap`）。  
  - 下方：**「创建实验」**按钮，与文案垂直排列、居中。
- **目的**：帮助商户理解功能并引导创建第一个实验。

### 7.2 文案与链接

- 变量名说明、对照组提示、自定义对象提示等见 4.1、4.2。
- 「了解详情」链接仅在选择「自定义」时展示，新开 tab 打开说明页（见 4.5）。

## 八、交互与体验规范（建议）

### 8.1 通用

- 列表行 hover 时行背景可弱高亮；复制按钮在悬停对应单元格时显示。
- 所有「取消」类按钮仅关闭弹窗或返回，不提交数据。
- 涉及状态变更（激活、暂停、结束、继续实验）时，操作成功后刷新当前视图或列表，保证状态与数据一致。

### 8.2 表单

- 必填项未填时，提交可标红并聚焦第一个非法项。
- 流量总和不等于 100% 时，可禁止提交或在表单项旁提示。
- 实验周期结束日期早于开始日期时，给予提示。

### 8.3 权限与多端（可选）

- 若有多角色，需约定谁可创建/编辑/激活/暂停/结束实验；列表与详情根据权限展示或隐藏按钮。
- 若支持多端，需约定路由与刷新策略（如从定价页带入创建后返回列表，列表需反映新建草稿）。

## 九、数据与接口约定（开发参考）

### 9.1 实验状态流转

```
草稿 --[激活实验]--> 运行中
运行中 --[暂停实验]--> 已暂停
运行中 --[结束实验]--> 已结束
已暂停 --[继续实验]--> 运行中
```

- 草稿、运行中、已暂停可编辑配置（已结束不展示编辑入口）。
- 结束实验时写入结束时间（如 `endedAt`），用于详情页「实验于 xxx 已结束」展示。

### 9.2 实验核心字段建议

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 实验唯一标识，列表展示且可复制 |
| name | string | 实验名称 |
| goal | string | 实验目标 |
| targetType | enum | 定价 / 价格表 / 自定义 |
| targetId / targetLabel | - | 定价或价格表时的标识与展示名；自定义时无 |
| customVarName | string | 实验对象为自定义时的变量名 |
| periodStart, periodEnd | date | 实验周期 |
| userScope | enum | 全部 / 定向 |
| userScopeRules | array | 定向时的维度与取值 |
| groups | array | 分组配置：name、traffic、及每组的定价/价格表 id 或自定义值 |
| status | enum | draft / active / paused / ended |
| endedAt | datetime | 结束时间，用于已结束展示 |
| createdAt, createdBy | - | 创建信息 |

### 9.3 用户分组数据

- 列表字段：uid、vid、group、assignedAt（分组时间）。
- 支持按 uid、vid 搜索与分页；支持单条修改用户分组并回写。

### 9.4 漏斗与转化率

- 漏斗 5 层：进入页面数、发起结账数、提交订单数、完成支付操作数、支付成功数。
- 转化率由上述数据计算得出，计算方式见 5.5；实验组与对照组的对比百分比由前后端约定（如相对增长率）。

## 十、UI 设计参考

- 状态标签：草稿（黄/橙）、运行中（绿）、已暂停（紫/蓝）、已结束（灰）。
- 主按钮：创建实验、激活实验、继续实验等用主色；次要操作（取消、返回）用次要按钮样式。
- 下拉菜单：操作按钮为次要样式，下拉项 hover 有背景变化；弹窗与现有后台风格统一（标题、正文、取消/主按钮）。
- 表格：表头与内容对齐清晰，转化率列与对比小字层级分明（字号、颜色区分）。
- 缺省模块：插图为辅助说明，不抢主文案；主文案与按钮居中、层次清晰。
