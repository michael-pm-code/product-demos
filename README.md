# Subotiz A/B 测试管理后台 - 产品原型

本原型用于 **产品设计** 与 **PRD（产品需求文档）** 参考，供开发实现商户端 A/B 测试管理功能时对照使用。

## 使用方式

1. 用浏览器直接打开 `index.html`，或使用本地静态服务（如 `npx serve .`）访问。
2. 默认进入「A/B 测试」实验列表页；无实验时显示空状态并引导创建。
3. 通过「创建实验」、列表行点击、Tab 切换等完成全流程体验。

## 功能结构概览

### 1. 主界面：实验列表

- **有实验时**：表格展示  
  **实验名称**、**创建时间**、**创建人**、**实验对象**、**实验时长**、**实验用户量**、**状态**（草稿 / 运行中 / 已结束）。  
  点击行进入该实验详情。
- **无实验时**：空状态文案 +「创建实验」按钮。
- 列表右上角提供「+ 创建实验」入口。

### 2. 创建 / 管理实验

#### 2.1 创建实验表单字段

| 字段 | 说明 | 备注 |
|------|------|------|
| 实验名称 | 展示给商户区分实验 | 必填 |
| 实验目标 | 如「提升定价页下单转化率」 | 选填 |
| 实验对象 | 可选「定价」或「价格表」 | 选填后展示下拉：定价展示「定价名称 + 定价 ID」，价格表展示「价格表名称 + 价格表链接」 |
| 实验周期 | 开始日期 ~ 结束日期 | - |
| 实验用户范围 | 全部用户 / 定向范围 | 定向时可添加多条规则：如国家、来源渠道、设备等 |
| 实验分组与流量 | 默认 1 个对照组，可再添加 1～5 个实验组；每组可配置流量占比，默认均分，总流量 100% | 总流量需为 100% |

#### 2.2 实验状态

- 创建完成后状态为 **草稿（draft）**。
- 在 **列表页** 或 **详情页** 操作「设为运行中」后变为 **运行中（active）**，实验才真正运行。
- 运行中可操作「结束实验」变为 **已结束（ended）**。

#### 2.3 实验详情 / 管理 Tab

- 展示完整实验配置：实验目标、实验对象、实验周期、用户范围、分组与流量。
- 提供「编辑配置」入口，用于修改上述配置（原型中仅作入口示意）。
- **用户实验分组数据** 表格：展示 **用户 Uid**、**用户 Vid**、**用户分组**、**分组时间**；支持通过下拉框 **手动修改** 某用户的实验分组。

### 3. 实验数据 Tab

- **头部汇总**：实验时长、参与用户数。
- **转化漏斗数据**（5 层）：  
  **进入页面数** → **发起结账数** → **提交订单数** → **完成支付操作数** → **支付成功数**。
- **数据表格**：  
  - 每一 **行** = 一个实验组（如对照组、实验组 A、实验组 B）。  
  - 列顺序：每个数据列后紧跟对应的转化率列。  
    - 发起结账数后 → **发起结账率**（= 发起结账数/进入页面数）  
    - 提交订单数后 → **提交订单转化率**（= 提交订单数/发起结账数）  
    - 完成支付操作数后 → **完成支付操作转化率**（= 完成支付操作数/提交订单数）  
    - 支付成功数后 → **支付成功转化率**（= 支付成功数/完成支付操作数）  
    - 最后一列 → **整体转化率**（= 支付成功数/进入页面数）  
  - 实验组的转化率下方展示**小字对比**：相对对照组的增长百分比（如 +10.2%、-3.1%）。

## 与前端 SDK 的配合（PRD 参考）

- 管理后台负责：实验的创建、配置、状态、用户分组与数据查看。
- 前端 SDK 负责：为访客生成唯一 id（如 vid）、按 id 做实验分组、拉取实验规则、向页面注入实验变量。
- 本原型仅覆盖 **商户端管理后台**，不包含 SDK 行为。

## 文件说明

- `index.html`：入口页与布局。
- `styles.css`：全局与各模块样式。
- `app.js`：路由、列表/创建/详情/数据视图渲染与交互（含模拟数据）。
