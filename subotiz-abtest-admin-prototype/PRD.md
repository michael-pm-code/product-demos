# Subotiz A/B 测试管理后台 - 产品需求文档（PRD）

**版本**：1.1（与当前 HTML 原型 demo 对齐）  
**目标读者**：开发、UI/UX 设计师  
**用途**：指导 A/B 测试管理后台的设计与开发实现  

---

## 一、文档说明

### 1.1 产品背景

A/B Test 系统用于帮助商户通过创建实验验证不同策略对转化率的影响，优化定价/价格表等配置，提升转化率与收入。本 PRD 描述商户端管理后台的全部功能、交互与业务规则。

### 1.2 功能范围

- **管理后台**：实验的创建、编辑、状态管理、用户分组管理、数据查看与 AI 解读。
- **前端 SDK**（不在本文档）：为访客生成唯一标识、实验分组、拉取实验规则、返回变量值；商户在站点集成 SDK 后根据返回值做不同策略处理。

### 1.3 术语

| 术语 | 说明 |
|------|------|
| 实验对象 | 实验作用的目标类型：定价、价格表、或自定义变量 |
| 对照组 | 不施加改动的分组，用于与实验组对比 |
| 实验组 | 施加不同策略的分组（如不同定价、不同文案等） |
| 实验 ID | 创建实验后由系统生成；自定义实验时，前端通过实验 ID 向 SDK 获取当前用户的分组与分组变量（下发值） |

## 二、全局与导航

### 2.1 布局结构

- **左侧导航**：固定侧栏，一级菜单为「数据」「商品」。
- **主内容区**：顶部页面标题（可带左侧返回按钮），下方为路由内容。

### 2.2 导航结构

| 一级菜单 | 二级菜单 | 是否可点击 | 路由/说明 |
|----------|----------|------------|-----------|
| 数据 | 交易概览 | 否 | 仅展示，对齐现有后台 |
| 数据 | 订阅概览 | 否 | 仅展示 |
| 数据 | 数据报表 | 否 | 仅展示 |
| 数据 | **A/B 测试** | 是 | 实验列表页 |
| 商品 | 商品 | 是 | 商品概览页 |
| 商品 | 定价 | 是 | 定价列表页 |
| 商品 | 价格表 | 是 | 价格表列表页 |

- 当前路由对应的二级菜单项高亮；所属一级菜单标题在高亮时也可做视觉强调。

### 2.3 页面标题与返回

- **A/B 测试列表**：标题「A/B 测试」，无返回按钮。
- **创建实验**：标题「创建实验」，标题左侧有「← 返回」按钮。
- **实验详情**：标题「实验详情」，标题左侧有「← 返回」按钮，点击返回实验列表。
- **商品 / 定价 / 价格表**：标题为对应名称，无返回按钮。

**创建实验页返回逻辑**：

- 若表单**未填写任何有效内容**：点击返回直接进入实验列表。
- 若表单**已填写内容**：点击返回弹出确认弹窗。  
  - 文案：「退出后填写的内容将丢失，是否继续？」  
  - 按钮：「取消」（关闭弹窗）、「继续退出」（关闭弹窗并进入实验列表）。

「已填写」判定范围建议：实验名称、实验目标与方案描述、实验对象、实验结束配置（与进入页默认值不一致时）、定向范围标签、任一分组的定价/价格表/自定义分组变量。

## 三、A/B 测试列表页

### 3.1 列表有数据时

- **表格列**（从左到右）：  
  实验名称、**实验 id**、创建时间、创建人（原型 demo 为邮箱）、实验对象、实验时长、实验用户量、状态。

- **实验名称列**：可展示名称 + 副文案（如定价/价格表名称或链接），支持多行。
- **实验 id 列**：展示实验唯一标识（如 `exp-001`），便于用标识定位问题。

**复制能力**（用于快速用实验标识定位问题）：

- 「实验名称」「实验 id」两列：鼠标悬停时在单元格内右侧展示「复制」按钮。
- 点击「复制」：将当前列对应内容写入剪贴板（名称列复制实验名称，id 列复制实验 id）。
- 复制成功后按钮可短暂变为「已复制」，约 1.5 秒后恢复为「复制」。
- 点击复制按钮不触发整行点击（不进入详情）。

- **状态**：展示为标签，可选状态见 3.3。
- **操作**：点击整行（除复制按钮外）进入该实验详情页。
- **右上角**：常驻「+ 创建实验」按钮，进入创建实验页。

- **搜索与筛选**：列表上方提供「搜索」输入框（占位「按实验名称或实验 ID 搜索」），支持按实验名称或实验 id 模糊匹配（不区分大小写）；与「实验状态」下拉筛选组合使用；分页保留搜索与筛选条件。

### 3.2 列表无数据时

- 列表区域展示**空状态**：
  - 图标 + 标题「还没有任何实验」。
  - 说明文案：创建第一个 A/B 实验，验证定价或价格表配置对转化率与收入的影响。
  - 「创建实验」按钮。
- 空状态下方仍展示「缺省状态」模块（见第七章），引导理解功能并创建实验。

### 3.3 实验状态枚举

| 状态值 | 展示文案 | 说明 |
|--------|----------|------|
| draft | 草稿 | 刚创建或未激活 |
| active | 运行中 | 正在放量 |
| paused | 已暂停 | 已暂停放量，可恢复 |
| ended | 已结束 | 实验结束 |

## 四、创建实验页

### 4.1 表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 实验名称 | 单行文本 | 是 | 供商户在后台区分实验，如「定价页标题 A/B 测试」 |
| 实验对象 | 单选 | 是 | 选项：定价、价格表、**自定义**；须先选择后配置分组变量 |
| 实验结束配置 | 达到用户数后停止、可选运行天数上限 | 是（人数）/ 否（运行天数为可选能力） | 见下文「实验结束配置」 |
| 实验用户范围 | 单选 + 条件配置 | 条件必填 | 选项：全部用户、**定向范围**。选择「定向范围」时须至少添加 **1 条**定向规则（维度如：国家、来源渠道、设备；取值为文本，以业务为准） |
| 实验分组与流量 | 表格 + 操作 | 见 4.2 | 除「分组说明」外，每组须填写名称、分组变量、流量 |
| 实验目标与方案描述 | 多行文本 | **否** | 说明测试目的与各组差异逻辑；原型占位为「实验目标与方案描述」 |

**实验结束配置**

- 标题下说明文案：实验优先按照达到测试用户数自动结束，可选设置达到运行时长也自动结束（实验用户越多结果越准确）。
- **实验达到 [ ] 用户数后自动停止**：同一行展示「实验达到 + 数字输入 + 用户数后自动停止」；人数上限 1000 万；须为 **501～10 000 000** 的整数（原型校验与标红见 4.1.1）。
  - 进入页面后获取店铺近 30 天日均用户数，按规则填入默认值：日均 &lt; 500 → 默认 2000；[500, 5000) → 默认 10000；[5000, ∞) → 默认 50000。
- **实验运行 [ ] 天后自动结束**：**可选**；默认不展开；在人数配置行下方提供较弱样式文字按钮「添加实验最大运行时长」，点击后展开「实验运行 + 数字输入 + 天后自动结束」，可「移除」收起。**展开后允许留空**（表示不限制最大运行天数，与原型一致）；若填写则须为 **1～365** 的整数。

**实验对象 = 定价 或 价格表时**：

- 不再在实验对象下展示二级下拉（不在此处选择具体定价/价格表）。
- 在「实验分组与流量」模块中，中间列表头随类型为「定价」或「价格表」（原型表头统一为「**分组变量**」列），每行下拉选择该组对应的定价或价格表。
- 对照组行下方展示提示：「商户站点前端集成的价格或价格表需要与对照组保持一致。」
- **价格表（原型演示）**：在「月度 / 年度」等可选项之外，增加 **1 条置灰不可选**的示例项，用于演示「已被其他实验占用」场景；`hover` 展示提示文案：`该选项已在实验「xxx」内`（正式环境由接口返回占用关系）。

**实验对象 = 自定义时**：

- 在「实验对象」选择框下方展示说明文案：「前端在实验节点调用 SDK 方法使用实验 ID 获取用户实验分组数据，商户根据返回值做不同实验策略处理。」
- 文案后增加文字链接「了解详情」，点击在**新标签页**打开「自定义 A/B Test 实验配置说明」页（见 4.4）。
- 在「实验分组与流量」中，「**分组变量**」列为文本输入，商户为各分组填写要下发给前端的值（如 `default`、`variant_a`）；「**分组说明**」列 **选填**。

### 4.1.1 提交前校验（与原型一致）

除以下字段外，**均视为必填**，未通过时对应控件 **红色描边**，并在**文档顺序中第一个出错控件**的下方展示提示文案 **「请完成此项配置」**，页面 **自动滚动**至该位置：

- **不校验必填**：分组说明（每行「分组说明」输入框）、**实验目标与方案描述**、**实验最大运行时长**（未展开则不校验；展开后允许整行留空；若填写则校验 1～365 整数）。

**须校验的内容包括**：

- 实验名称非空；实验对象已选择；实验达到用户数合法（501～10 000 000）；  
- 用户范围为「定向范围」时，至少一条定向标签；  
- 每组：**实验分组名称**非空（对照组只读已有名称）；**分组变量**已填（下拉有值或自定义非空）；  
- 所有分组 **流量百分比之和 = 100%**（否则所有流量输入框标红）。

用户修改表单输入时，原型会清除当前标红状态，需再次提交触发校验。

### 4.1.2 提交与跳转

- **保存为草稿**：通过校验后提交；原型为提示「已保存为草稿」并跳转实验列表（正式环境可改为留在当前页）。
- **取消 / 返回**：与 2.3 一致；未实现「取消」单独二次确认时，以产品为准。

### 4.2 实验分组与流量

- **表头**：**实验分组名称** | **分组变量** | **分组说明** | 流量 % | 操作  
  - 「分组变量」列：随「实验对象」为定价 / 价格表 / 自定义，展示为下拉或文本输入（语义同「为该组选择的定价/价格表或自定义下发值」）。
- **行**：
  - 第一行固定为**对照组**，**实验分组名称**只读（默认「对照组」）。
  - 可再添加 1～5 个**实验组**（即总组数 2～6）。
  - 每行：实验分组名称、分组变量、分组说明（选填）、流量 %（数字输入）。
- **删除实验组**：非对照组且实验组多于 1 个时可删除；原型使用 **垃圾桶图标按钮**（非「删除」文字），`title`/`aria-label` 为「删除该实验组」。
- **流量规则**：
  - 各分组流量百分比之和须为 100%。
  - **添加实验组**时：新增一行，并将**所有分组**的流量自动重新均分，保证总和 100%（如 4 组则 25%、25%、25%、25%）。
  - 商户可手动修改各组分比；提交时与 4.1.1 一致校验总和。
- **对照组提示**：仅在实验对象为定价或价格表时显示，见 4.1。

### 4.3 从定价/价格表页带入

- 在「定价」或「价格表」列表中，每行提供「创建 A/B 实验」按钮。
- 点击后跳转到创建实验页，并**自动**：
  - 将「实验对象」设为对应类型（定价 或 价格表）。
  - 将**对照组**的「分组变量」设为当前行对应的定价或价格表（若该价格在原型中被演示为「已占用」置灰项，则**不会**强行选中，对照组需用户另选）。

### 4.4 自定义 A/B Test 实验配置说明页

- **入口**：创建实验页 - 实验对象为「自定义」时，说明文案旁的「了解详情」。
- **打开方式**：新标签页打开，路由建议独立（如 `/docs/custom-abtest-config` 或 hash 等价）。
- **内容**：说明文档，包含但不限于：
  - 如何通过实验 ID 调用 SDK；
  - 如何为各分组填写「分组变量」（下发值）；
  - 前端如何在实验节点根据返回值做不同策略。

## 五、实验详情页

### 5.1 页面结构

- **顶部**：左侧为 **实验名称** 与 **时间信息**（见下），右侧为操作区（见 5.2）。
- **Tab**：**实验详情 / 管理**（默认）、**实验数据**。切换 Tab 不离开当前实验。

**标题区时间（与原型一致）**

- **草稿**：不展示开始/结束时间行。
- **非草稿**：展示 **实验开始时间**（无数据时展示「—」）；首次从草稿激活/继续实验时写入开始时间（原型为当前时间 Mock）。
- **已结束**：在「实验开始时间」旁追加 **结束时间**（含手动终止或自动结束；字段如 `endedAt`）。

### 5.2 头部操作区（按状态展示）

- **编辑配置**  
  - 仅当状态**不是**「已结束」时展示。  
  - 点击进入编辑（原型为提示入口；正式环境为与创建实验字段一致的编辑表单）。

- **状态标签**  
  - 始终展示：草稿 / 运行中 / 已暂停 / 已结束。

- **草稿**  
  - 展示「**操作**」下拉：**激活实验**、**结束实验**。  
  - **激活实验**（见 5.3.2）：若实验对象为价格表，且系统中已存在其他**非已结束**的价格表实验，则**拦截**（仅提示，不激活）。  
  - **结束实验**：**无需二次确认**，直接变为「已结束」并写入结束时间。

- **已暂停**  
  - 展示按钮「**继续实验**」。点击后将状态改为「运行中」；若无开始时间则写入（同激活）。

- **运行中**  
  - 展示「**操作**」下拉：**暂停实验**、**结束实验**（见 5.3）。

### 5.3 暂停 / 终止确认弹窗与激活拦截

#### 5.3.1 暂停实验

- **标题**：提示  
- **正文**：暂停状态下将无新用户进入实验，已进入实验用户不受影响  
- **按钮**：取消、**暂停实验**（确认后状态改为已暂停并刷新）

#### 5.3.2 结束实验（运行中 / 已暂停，或草稿已改为「结束」见 5.2）

- **非草稿**时点击「结束实验」须二次确认：  
  - **标题**：提示  
  - **正文**：终止实验操作不可逆，终止实验后将无新用户进入实验  
  - **按钮**：取消、**终止实验**（确认后状态改为已结束，写入 `endedAt`）

#### 5.3.3 激活实验（价格表互斥，原型）

- 当实验对象为**价格表**且已存在另一条 **target = 价格表** 且 **status ≠ ended** 的实验时：  
  - **标题**：提示  
  - **正文**：当前已有正在进行的「价格表」实验，同一时间只能运行一个价格表实验  
  - **按钮**：仅 **确定**（关闭弹窗，**不执行激活**）

### 5.4 Tab：实验详情 / 管理

- **元信息**（配置信息上方单独一块，2×2 网格）：  
  **实验创建时间**、**创建人**、**实验更新时间**、**更新人**（原型 demo 创建人/更新人为邮箱格式 Mock）。

- **实验配置信息**（只读展示）：  
  实验目标与方案描述、实验对象（类型 + 名称/ID 或链接）、实验结束配置、实验用户范围（全部或定向规则列表）、实验分组与流量（组名 + 流量占比）。

- **用户实验分组数据**  
  - **表格列**：experiment uid、customer id、payer id、IP 地址、用户分组、分组时间。  
  - **多值（原型）**：customer id、payer id 可为多个值；多值时默认展示一项，通过下拉切换查看其余值；搜索若命中某一值，该行下拉默认选中该匹配值。  
  - **搜索**：表格上方右侧提供搜索框；**按回车**或输入框**失焦**后执行筛选（避免每输入一字即刷新），支持对 experiment uid、customer id、payer id、IP 等字段模糊查询（不区分大小写）。  
  - **分页**：数据量较大时分页展示（如每页 10 条），底部展示「共 x 条」及上一页/下一页（或页码），切换分页时保留搜索条件。  
  - **修改分组**：每行「用户分组」为下拉框，商户可手动将某用户改为其他分组，保存后生效（具体保存策略由后端约定）。

### 5.5 Tab：实验数据

- **「实验数据」Tab 提示（仅已结束实验，原型）**  
  - 在 Tab 文案「实验数据」后展示 **?** 提示图标；悬停展示说明：实验结束后不再进入新用户；已进入实验用户的长周期或延迟反馈类数据（例如续费率、退款率等）将持续更新。

- **汇总**：实验时长、参与用户数（可选更多指标）。

- **转化数据（5 层漏斗，与原型列名一致）**  
  - 指标顺序：浏览用户数 → 发起结账用户数 → 提交订单用户数 → 完成支付操作用户数 → 支付成功用户数。  
  - 每行一个实验组；每个绝对值指标列后紧跟**转化率列**（浏览结账率、订单提交率、支付操作完成率、支付成功率、**浏览转化率**）。  
  - **实验组**转化率下方展示相对对照组的变化百分比（绿/红）；**浏览转化率**旁对实验组展示置信度（原型 Mock）。

- **收入与风险数据（原型）**  
  - 表格列包括：支付用户数、支付订单数、GMV、ARPU、**订阅续费率**、退款率、拒付率等；实验组部分指标下展示相对对照组变化及置信度（规则见原型）。  
  - **「订阅续费率」表头**旁设提示图标，悬停说明：**关于续费数据的特别说明**——续费发生在订阅周期结束时；当前续费率仅基于已到期用户样本；建议在实验结束后再经过至少一个完整订阅周期后再做最终决策（示例：试用 7 天则延长约 7 天观察，月付约 30 天等）。

- **AI 数据解读**  
  - 收入与风险表格下方为「AI 数据解读」模块，基于 Mock 数据生成简要结论（正式环境由接口返回）。

## 八、交互与体验规范（建议）

### 8.1 通用

- 列表行 hover 时行背景可弱高亮；复制按钮在悬停对应单元格时显示。
- 所有「取消」类按钮仅关闭弹窗或返回，不提交数据。
- 涉及状态变更（激活、暂停、结束、继续实验）时，操作成功后刷新当前视图或列表，保证状态与数据一致。

### 8.2 表单

- 创建实验：未通过校验时，对应控件标红，在**第一个出错项**下方展示「请完成此项配置」，并滚动到视区内（见 4.1.1）。
- 流量总和不等于 100% 时，所有流量输入框标红并纳入上述校验。
- 实验结束配置中人数、可选运行天数的规则见 4.1、4.1.1。

### 8.3 权限与多端（可选）

- 若有多角色，需约定谁可创建/编辑/激活/暂停/结束实验；列表与详情根据权限展示或隐藏按钮。
- 若支持多端，需约定路由与刷新策略（如从定价页带入创建后返回列表，列表需反映新建草稿）。

## 九、数据与接口约定（开发参考）

### 9.1 实验状态流转

```
草稿 --[激活实验]--> 运行中
运行中 --[暂停实验]--> 已暂停
运行中 --[结束实验]--> 已结束
已暂停 --[继续实验]--> 运行中
```

- 草稿、运行中、已暂停可编辑配置（已结束不展示编辑入口）。
- 结束实验时写入结束时间（如 `endedAt`），用于详情页「实验于 xxx 已结束」展示。

### 9.2 实验核心字段建议

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 实验唯一标识，列表展示且可复制 |
| name | string | 实验名称 |
| goal | string | 实验目标 |
| targetType | enum | 定价 / 价格表 / 自定义 |
| targetId / targetLabel | - | 定价或价格表时的标识与展示名；自定义时无 |
| endUserStop | int | 达到该参与用户数后自动结束（须 &gt; 500） |
| endRunDays | int | 运行该天数后自动结束（1～365） |
| userScope | enum | 全部 / 定向 |
| userScopeRules | array | 定向时的维度与取值 |
| groups | array | 分组配置：name、traffic、及每组的定价/价格表 id 或自定义值 |
| status | enum | draft / active / paused / ended |
| endedAt | datetime | 结束时间；详情页「已结束」时与时间区展示 |
| startedAt | datetime | 实验开始时间（非草稿且已开始时） |
| createdAt, createdBy | - | 创建时间与创建人（展示可用邮箱） |
| updatedAt, updatedBy | - | 最后更新时间与更新人（展示可用邮箱） |

### 9.3 用户分组数据

- 列表字段：experiment uid（`experimentUid`）、customer id（`customerIds` 数组，可多个）、payer id（`payerIds` 数组，可多个）、IP 地址（`ip`）、用户分组（`group`）、分组时间（`assignedAt`）；行主键可用 `rowKey`。
- 支持按 experiment uid、任一 customer id、任一 payer id、IP 搜索与分页；支持单条修改用户分组并回写。

### 9.4 漏斗、收入与转化率

- **漏斗 5 层（原型列名）**：浏览用户数、发起结账用户数、提交订单用户数、完成支付操作用户数、支付成功用户数；转化率与计算公式见界面表头 hover 及 5.5。
- **收入与风险**：含 GMV、ARPU、订阅续费率、退款率、拒付率等；续费率指标需结合长周期与样本说明（见 5.5 表头提示）。
- 实验组与对照组的对比百分比由前后端约定（如相对增长率）；部分指标可展示置信度（原型为 Mock）。

## 十、UI 设计参考

- 状态标签：草稿（黄/橙）、运行中（绿）、已暂停（紫/蓝）、已结束（灰）。
- 主按钮：创建实验、激活实验、继续实验等用主色；次要操作（取消、返回）用次要按钮样式。
- 下拉菜单：操作按钮为次要样式，下拉项 hover 有背景变化；弹窗与现有后台风格统一（标题、正文、取消/主按钮）。
- 表格：表头与内容对齐清晰，转化率列与对比小字层级分明（字号、颜色区分）。
- 缺省模块：插图为辅助说明，不抢主文案；主文案与按钮居中、层次清晰。
